    \section{背景}
        2040年問題として、介護業界など、ブルーカラーの仕事需要が向上すると予想される。そこで、自立型エージェント(IOT)としてふるまえる装置が必要だ。
    
    \section{最終目標}
        従来のLLMは意味空間を用いて高度な推論が可能だが、実世界での実行能力に関して、乏しい。しかしこれを解決するため、巨大な再学習を行うことは計算資源として非効率であり、反射神経的な学習になりがち。そこで、RAGやECSを用いて、環境情報を効率的に処理し、LLMの実行能力を向上させることが最終目標である。\\
        * 本研究は基本的にLLMの中身を改良するものではなく、LLMを内包する制御構造を最適化する研究である。\\
        \begin{itemize}
            \item ECS:高次元な環境を、LLMが解釈可能な形に構造化する \\
            \item メタプロトコル:interfaceがテキストのみで定義されても実行可能にする機構
        \end{itemize}
        
    \section{なぜECSなのか}
    \begin{itemize}
        \item 水平スケーリングがたやすい
        \item 現実という文脈をすべて階層構造に置き換えるのでようはトークンレベル、ベクトルとして扱える。なので、LLMとの親和性が高い
        \item メタプロトコルによって、LLMとECSのやりとりを定義することで、柔軟なシステム構成が可能になる。
        \item 人間にとっても扱いやすい、可視化を通してdebugを行いやすい
        \item 疎結合であるため、システムの拡張や変更が容易になる。
        \item メタプロトコル＋ECSによって、LLMの推論を拘束することができる。
    \end{itemize}

    \section{システム構成}
    \begin{itemize}
        \item Core（API的なものともいえる疎結合なので）: Rustによる仲介処理
        \item ECS: Web上で動作するエンジン
        \item RAG: qdrantを用いたベクトルデータベース
        \item メタプロトコル: LLMとECSのやりとりを定義するプロトコル
    \end{itemize}
    \begin{itemize}
        \item 本研究ではLLMを確率的なサイコロではなく、拘束条件付き最適化問題として再定義する。
        \item 物理エンティティの状態Sを独立したコンポーネントとその追従したデータの集合として定義する。
        
            S = {C1, C2, ..., Cn} ここで、Ciはコンポーネントiを表す。 \\
    例えば、RAGを用いて、過去の状態や外部情報をベクトルデータベースに格納し、LLMが必要な情報を迅速に取得できるようにすることができる。
    また、ECSを用いて、物理エンティティの状態を階層構造で管理し、LLMが状態の変化を効率的に処理できるようにすることができる。
    これにより、LLMの推論能力を向上させ、実世界の実行能力を向上させることができる。
    さらに、メタプロトコルを用いて、LLMとECSのやりとりを定義することで、柔軟なシステム構成が可能になる。
    例えば、LLMがECSに対して状態の更新を要求する場合、メタプロトコルを用いて、その要求を定義することができる。
    これにより、LLMの推論を拘束することができ、システムの拡張や変更が容易になる。 \\
    デバイス自体をエンティティと考えて、1:N、N:Nというネットワーク構造をとるのを想定している。
    (１つのECSを複数台で管理など)
    \end{itemize}

    例えるならば、サイコロの２に変幻可能なおもりをつけるようなものである（拘束）。すると５が出やすくなるのは当たり前、これを確率論とみることもできるが、物理現象的に説明をすることができる。
    ではこのおもりの形をどのようにしたら、５の側面部分の目が出るのか、というのがこの研究の主題である。
    拘束をする理由は、BIGTECのような集中型のAIよりも、囲い込んだ方が、安全かつ低コスト。
    既存のAIロボット（Spot等）は「転びそうになったら踏ん張る」という反射神経を数百万回のシミュレーションで学習させます。対してあなたのシステムは、**「転ばないための物理的拘束条件」**をLLMが事前に論理として把握し、ECSがそれを保証する。つまり、「練習」なしで「正解」を導き出す。
